# DBM capability map

This matrix is the source of truth for how DBM capabilities mature across the process-first reset roadmap.

| Capability | Architectural role | `R1` process/stage designer and actual form render | `R2` DBMScript and action foundation | `R3` back-office runtime | `R4` back-office operations | `R5` portal runtime and return path | `R6` reuse, templates, artefacts, and documents | `R7` platform tooling and ALM | `R8` enterprise maturity | `R9` AI-assisted platform |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Process portfolio | Authoritative business-cycle model | Define `mainProcessId`, `processes[]`, sub-process lanes, and stage spans | Expose action attachment points | Persist runtime instance state | Add operational controls | Project portal-safe state | Add reusable templates and base flows | Package and sync process assets | Govern portfolios at scale | Draft and review process portfolios |
| Stage spans | Visual and semantic alignment across processes | Support whole-stage, multi-stage, and fractional main-stage spans | Attach action hooks to span-aware stages | Execute transitions using span-aware state | Analyse SLA and routing by span | Render portal-safe spans | Reuse span patterns | Validate span migrations | Simulate span behaviour | Suggest span improvements |
| Rendered form experience | Business-user form surface | Actual model-driven form render | Expose dynamic button and template hooks | Execute form behaviour and state changes | Add validations, tasks, and support status | Add actual portal rendering later | Generate form-related artefacts | Package form/runtime assets | Optimise form behaviour | Suggest form improvements |
| DBMScript/actions | Executable logic substrate | Model feature hooks only | JavaScript first action, VM, DBM Object composition, templates, WYSIWYG notification templates, query editor hooks, version history, and test case support | Execute triggers in back office | Drive routing, SLA, notifications, jobs, and validations | Run portal-safe actions | Power cloning, generation, and documents | Sync/version/package scripts and objects | Govern and trace scripts | Suggest/review logic |
| Notifications | Communication capability | Define hooks and status semantics | Table row templates plus send actions | Trigger from stage/runtime events | Operational notification handling | Portal-safe notification projection | Reusable templates | Package/version templates | Audit and optimise notification patterns | Draft notification text |
| Routing and work | Operational assignment | Define hooks | Define action/template support | Persist process sessions, switching, owner/scope state, and transition mode | Deliver routing, tasks, work sequences, and SLA/KPI | Expose portal-safe status only | Reuse routing templates | Package operational assets | Optimise work distribution | Suggest routing policies |
| Portal continuity | External user experience | Portal contract only | Portal-safe action contract | Persist portal-status projection fields | Support send-back and operational status | Actual portal rendering and return path | Reuse portal templates | Package portal assets | Govern external projections | Draft portal copy and checks |
| Generated artefacts | Repeatable business outputs | Define artefact hooks | Define templates/actions | Trigger from runtime | Operationalise generation | Expose portal-safe outputs | Deliver artefacts, documents, cloning, numbering | Package and sync artefacts | Govern generated artefacts | Suggest artefacts |
| Platform ALM | Durable delivery | Keep contracts portable | Version scripts/actions/objects | Version runtime assets | Version operational assets | Version portal assets | Version reusable artefacts | Deliver DBM Manager, DBM Solution, DBM Tree, XrmToolBox playground, source sync, post-deploy scripts, and auto-integration discovery | Govern drift and policy | Review generated changes |
| Enterprise insight | Support and optimisation | Capture explainability hooks | Trace action execution | Trace runtime transitions | Deliver history/audit and support surfaces | Trace portal paths | Trace generated artefacts | Trace ALM operations | Deliver simulation, replay, drift control, observability, and optimisation | Summarise and recommend |
