# DBM capability map

This matrix is the source of truth for how DBM capabilities mature across the process-first reset roadmap.

| Capability | Architectural role | `R1` process/stage designer and actual form render | `R2` DBMScript and action foundation | `R3` back-office runtime | `R4` back-office operations | `R5` portal runtime and return path | `R6` reuse, templates, artefacts, and documents | `R7` platform tooling and ALM | `R8` enterprise maturity | `R9` AI-assisted platform |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Process portfolio | Authoritative business-cycle model | Define `mainProcessId`, `processes[]`, nested child process definitions, and stage-owned child process links | Expose action attachment points and compile published snapshots from authoring rows | Persist runtime instance state and child process instances from published snapshots/definitions only | Add operational controls | Project portal-safe state | Add reusable templates and base flows | Package and sync process assets through compiled published snapshots | Govern portfolios at scale | Draft and review process portfolios |
| Collaborative authoring | Multi-user authoring foundation | Single-user designer proof only | Dataverse-normalised authoring rows, private drafts, designer-session presence, granular edit leases, rowversion/ETag guards, and Power Apps Code Apps proof | Runtime consumes published snapshots/definitions only and never drafts, designer sessions, or locks | Routing, SLA, notifications, validations, and operational configs become locked/drafted authoring rows | Portal authoring respects published projection definitions only | Template and artefact authoring uses the same draft/lock lifecycle | Export/import compiled published snapshots plus optional source-normalised artefacts with conflict reporting | Govern sessions, locks, drafts, conflicts, and authoring policy at scale | Assist users without bypassing sessions, locks, drafts, or publish review |
| Stage child process links | Parent/child process hierarchy anchor | Support `childProcessRefs[]`, parent stage context, and blocked/awaiting-child indicators | Attach action hooks to linked stages | Spawn child process instances and unlock parent stages on completion | Analyse SLA and routing by parent/child context | Render portal-safe child process context | Reuse child process patterns | Validate child-link migrations | Simulate child process behaviour | Suggest child process improvements |
| Rendered form experience | Business-user form surface | Actual model-driven form render | Expose dynamic button and template hooks | Execute form behaviour and state changes | Add validations, tasks, and support status | Add actual portal rendering later | Generate form-related artefacts | Package form/runtime assets | Optimise form behaviour | Suggest form improvements |
| DBMScript/actions | Executable logic substrate | Model feature hooks only | JavaScript first action, VM, DBM Object composition, templates, WYSIWYG notification templates, query editor hooks, version history, test case support, private drafts, and granular locks | Execute published triggers in back office | Drive routing, SLA, notifications, jobs, and validations | Run portal-safe actions | Power cloning, generation, and documents | Sync/version/package scripts and objects with conflict reporting | Govern and trace scripts | Suggest/review logic |
| Notifications | Communication capability | Define hooks and status semantics | Table row templates plus send actions with authoring-row lifecycle | Trigger from stage/runtime events | Operational notification handling through first-class locked/drafted template rows | Portal-safe notification projection | Reusable templates | Package/version templates and source-normalised artefacts | Audit and optimise notification patterns | Draft notification text |
| Routing and work | Operational assignment | Define hooks | Define action/template support and policy authoring row shape | Persist process sessions, switching, owner/scope state, and transition mode | Deliver routing, tasks, work sequences, and SLA/KPI through first-class locked/drafted policy rows | Expose portal-safe status only | Reuse routing templates | Package operational assets and report source-sync conflicts | Optimise work distribution | Suggest routing policies |
| Portal continuity | External user experience | Portal contract only | Portal-safe action contract | Persist portal-status projection fields | Support send-back and operational status | Actual portal rendering and return path | Reuse portal templates | Package portal assets | Govern external projections | Draft portal copy and checks |
| Generated artefacts | Repeatable business outputs | Define artefact hooks | Define templates/actions | Trigger from runtime | Operationalise generation | Expose portal-safe outputs | Deliver artefacts, documents, cloning, numbering | Package and sync artefacts | Govern generated artefacts | Suggest artefacts |
| Platform ALM | Durable delivery | Keep contracts portable | Version scripts/actions/objects and establish compiled snapshot boundaries | Version runtime assets from published definitions | Version operational authoring rows | Version portal assets | Version reusable artefacts | Deliver DBM Manager, DBM Solution, DBM Tree, XrmToolBox playground, source sync, compiled published snapshots, source-normalised artefacts, conflict reporting, post-deploy scripts, and auto-integration discovery | Govern drift and policy | Review generated changes |
| Enterprise insight | Support and optimisation | Capture explainability hooks | Trace action execution | Trace runtime transitions | Deliver history/audit and support surfaces | Trace portal paths | Trace generated artefacts | Trace ALM operations | Deliver simulation, replay, drift control, observability, and optimisation | Summarise and recommend |
