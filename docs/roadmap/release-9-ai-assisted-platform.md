# Release 9: AI-assisted platform

## Goal

Add AI assistance after the basic product is stable, reviewable, testable, and useful without AI.

## Feature set and deliverables

- Requirement-to-process draft generation.
- Logic and condition suggestions.
- Missing-step and missing-data analysis.
- Test scenario generation.
- Optimisation recommendations.
- Human review gates for all generated output.
- Traceable AI explanations tied back to DBM model elements.

## Stages

### R9.1 AI guardrails and evidence model

Output:
- safe AI boundary for DBM

Must include:
- no unreviewed production mutation
- source attribution
- review workflow
- generated-output audit trail

### R9.2 Draft generation

Output:
- AI-assisted draft process creation

Must include:
- requirement ingestion
- draft process portfolio
- draft stages and sub-processes
- draft conditions and actions marked as unapproved

### R9.3 Review and optimisation assistance

Output:
- AI suggestions for already-authored DBM assets

Must include:
- missing-step analysis
- missing-data analysis
- condition suggestions
- test scenario suggestions
- optimisation suggestions

### R9.4 Adoption hardening

Output:
- AI assistance ready for controlled use

Must include:
- feedback loop
- accuracy review
- prompt/version governance
- support runbooks

## Exit criteria

- AI assists the user without replacing the DBM model, designer, runtime, tests, or human review.
